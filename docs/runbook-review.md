# スマホ承認 運用手順書（レビューRunbook） — Issue #11

担当者が**スマホで記事をチェックして公開するだけ**で運用が回る手順。
ツール側が下書きを積み、担当者はそれを確認→公開（または却下）する。**追加開発は不要**で、WordPress公式アプリを使う。

## 役割分担
| 役割 | やること | 必要権限 |
|------|---------|---------|
| 自動化ツール（無人） | 記事生成→もしもリンク挿入→**下書き**投稿 | （APIユーザー） |
| チェック担当者 | スマホで下書きを確認→**公開** or 却下 | WordPress「編集者」 |
| 管理者 | 担当者アカウント発行・キュー投入・障害対応 | 管理者 |

## 初期セットアップ（管理者が一度だけ）
1. **担当者用アカウントを作成**
   WordPress管理画面 → ユーザー → 新規追加 → 権限グループ＝**「編集者（Editor）」**
   （編集者は記事の編集・公開はできるが、テーマ/プラグイン/設定は触れない＝安全）
2. 担当者のスマホに **WordPress公式アプリ**（iOS/Android）をインストール
   - アプリ → ログイン → サイトアドレスに `https://ouchibase.com` → 担当者アカウントでログイン
3. 担当者に本書と [チェックリスト](./checklist-review.md) を共有

## 日次レビュー手順（担当者・スマホ）
所要：1記事あたり数分。1日分（例15件）で30〜60分が目安。

1. WordPressアプリを開く → **「投稿」→「下書き」**タブ
2. 下書きを1件開く
3. **プレビュー**で表示を確認（アプリ右上のプレビュー）
   - ここで初めて**楽天カード（もしもリンク）が実際に描画**される（ローカルや編集画面では出ない）
4. [チェックリスト](./checklist-review.md) に沿って確認
5. 判断：
   - **問題なし → 「公開」** をタップ
   - **軽微な誤り → その場で修正して公開**（誤字・言い回し等）
   - **事実誤り/重大 → 「ゴミ箱へ」**（特に「どこの国の企業か」が怪しい場合）
6. 次の下書きへ

## 公開後（任意）
- 公開された記事は自動でインデックス申請（Issue #17で整備予定）。
- 実績（流入・成果）は Issue #20 のモニタリングで追跡。

## トラブル時
| 症状 | 対処 |
|------|------|
| 下書きが増えない | 管理者へ連絡（バッチ未実行/キュー空/APIエラーの可能性、Issue #21の通知で検知） |
| 楽天カードが出ない | プレビューで確認（編集画面では出ない仕様）。それでも出ない場合はリンク差し替え |
| 企業情報が怪しい | 公開せずゴミ箱へ。`company_hint` 改善は管理者へフィードバック |

## 補足
- 既定が「下書き」なのは、AI生成の事実誤りを**人が必ず一度見る**ため。
- 将来、チェックの一部は自動QA（Issue #16）で機械化し、担当者の負荷をさらに下げる。
- 専用のワンタップ承認画面が必要になれば Issue #12 で用意する。
