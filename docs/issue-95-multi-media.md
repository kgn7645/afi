# 複数媒体への並行量産（#95）

複数のWordPressサイト（将来はnote/Xも）へ**並行で自動生成・量産**する。
現状の単一サイト(おうちベース)を壊さず、媒体を足せる「媒体プロファイル」構成にする。

---

## 全体像

```
媒体プロファイル(Media)  ── 各媒体に WP接続 / コンセプト / E-E-A-T / カテゴリ /
                              クロール語 / アフィリタグ を束ねる
   ├ #1 ouchibase   ← config.yaml トップレベル（既存・常に有効）
   ├ #2 （新媒体）   ← config.yaml `media:` に追加 + env に WP2_* 接続情報
   └ #N ...
        │
   並行バッチ ── 媒体ごとに「クロール→候補→承認→生成→投稿」を独立に回す
        │
   投稿先は contextvar(active media) で切替（wordpress.py が参照）
```

### 機密情報の置き場所（重要）
- **WP接続（URL/ユーザー/アプリパスワード）= env**。config.yaml はコミットされるので置かない。
- 媒体ごとに **env_prefix** で分ける:
  - 媒体#1: `WP_BASE_URL` / `WP_USERNAME` / `WP_APP_PASSWORD`（既存）
  - 媒体#2: `WP2_BASE_URL` / `WP2_USERNAME` / `WP2_APP_PASSWORD`
- 非機密（媒体名・コンセプト・クロール語・カテゴリ）は config.yaml `media:` ＋ 設定画面で編集可。

## ✅ あなたの作業: 2サイト目（WordPress）の構築手順

### 1. ドメインを用意
- 新ジャンル用のドメインを取得（お名前.com / Xserverドメイン 等）。
- **おすすめ: 既存と同じ Xserver アカウントに新ドメインを追加**。理由:
  - 追加費用が安く、サーバーは共用（管理が1か所）。
  - **日本IPを維持**できる（Amazon商品ページ取得は日本IPで安定。海外IPだと弾かれる）。

### 2. WordPress をインストール
- Xserver パネル → 「WordPress簡単インストール」→ 新ドメインを選択 → インストール。
- テーマ・パーマリンク（投稿名推奨）・サイト名を設定。

### 3. REST API用の「アプリケーションパスワード」を発行
- 新WPの管理画面 → ユーザー → プロフィール → 一番下「アプリケーションパスワード」。
- 名前（例: `affiliate-tool`）を入れて発行 → 表示される **xxxx xxxx xxxx xxxx** を控える
  （これがAPI投稿用パスワード。ログインパスワードとは別）。

### 4. 接続情報を両ホストの env に設定
Xserver（生成）と Render（リライト）の両方に、媒体#2の接続情報を追加:
```
WP2_BASE_URL=https://新ドメイン
WP2_USERNAME=管理ユーザー名
WP2_APP_PASSWORD=発行したアプリパスワード（空白は詰めずそのままでOK）
```
- Xserver: `~/afi/.env` に追記。
- Render: ダッシュボード → Environment に3つ追加。

### 5. 媒体を登録（config.yaml `media:` を有効化）
`config.yaml` の `media:` にコメントで用意してあるテンプレを参考に1媒体追加し、`enabled: true`、
`env_prefix: "WP2_"`、`concept` / `candidates.keywords` 等をその媒体のジャンルに合わせて記入。

### 6. 連絡をください
接続情報を設定し終えたら教えてください。フェーズ1〜4（投稿先切替・並行バッチ・媒体別キュー・
管理UI）を実装し、**実サイトで検証しながら**量産を立ち上げます。

> ⚠️ 接続情報（特にアプリパスワード）はチャットに貼らず、各ホストの env に直接設定してください。
> 設定済みかどうかの確認だけこちらで行います（値は伏字で扱います）。

## note / X（別トラック）
- **note**: 既に下書き連携あり（`NOTE_SESSION`）。媒体プロファイルに `note_enabled` を持たせ、
  WP媒体と同様に媒体単位で下書き作成する想定。
- **X(旧Twitter)**: 記事公開時に要約＋リンクを自動ポスト（X API v2 / 要開発者アカウント）。
  流入多様化(C)の一環。WP量産が回ってから着手を推奨。

## 設計メモ（実装者向け）
- 投稿先切替は `core/media.py` の contextvar (`set_active`/`active`/`wp_target`) を使い、
  `wordpress.py` の base_url/auth を `media.wp_target()` 経由にする（未設定時は従来 env=後方互換）。
- `get_rules()` 由来の concept/eeat/candidates は媒体別に上書きが要る → pipeline/batch に
  「アクティブ媒体のルール」を渡すヘルパを用意（媒体の eeat/candidates でトップレベルを部分上書き）。
- overrides の媒体別キー（`_manual_pending` 等）は `media:<id>:_manual_pending` のように
  名前空間化し、媒体間でキューが混ざらないようにする。
- 並行といっても Gemini はレート制御あり。媒体ループは直列でも可（cron間隔内で順番に）。
  まずは「媒体ごとに per_run 件ずつ順番に生成」で十分。
