## 2. 収益化モデル（最重要・事業判断）

**この10アカ中、商品アフィリ（楽天/Amazon/もしも型）で稼ぐアカは事実上ゼロ。**
占いニッチの収益化は3類型に集約:

1. **LINE誘導 → 有料鑑定**（reikan, yui）… 「初回無料」をフックにLINE登録、収益はThreads外で発生
   → Threadsがクリック数を返さない問題を構造的に回避できる
2. **外部サービス**（tu36159=ココナラ / shica=note / tokiyomi=lit.link投げ銭）
3. **占い師向け講座/コンサル**（k.styledesign, yuriko, corrida）… B2B（占い師を顧客化）

**🟢 唯一の商品アフィリ成功例 = `fuchu_ataru`（A8.net `px.a8.net/svt/ejp`）**
→ 電話占い・占いアプリ系のASP案件を貼る唯一の例。**スケール可能な唯一のアフィリ両立モデル。**

> **示唆**: 既存 `affiliate-automation`（物販リンク量産）はこのニッチにそのまま乗らない。
> 3アカ目は**収益を「電話占い/占いアプリASP案件（A8・afb・もしも）」へ切替**が前提。

## 4. バズ文体の共通則

- **超短文＋絵文字アクション**（「🌹おきなはれ」「🐱を置いてみて」）でコメント欄を埋めさせ、アルゴに乗せる
- **断定×安心**（「私は嘘は言いません！！」「良いことしかないから」）
- **限定・希少**（「週10人限定」「今だけ扉が開く」）
- **共感ストーリー人格**（fuchu「男に2度騙されて占星術に救われた女・シングルマザー」）＝最もアフィリと両立
- **二人称の直接呼びかけ**（「このメッセージが目に留まった今のあなたへ」maho）
- **実績バッジをbio先頭**（reikan「鑑定歴10年・5,000人」／yui「年間1万人」）

## 5. 投稿頻度（実観測）

- reikan.moon: 同日 08:45/10:45/12:45/14:45 …**約2時間おき＝1日4〜8本**の高頻度
- fuchu_ataru: 12星座を毎朝バースト＋雑談＝**1日10本超**
- 他は1日0.2〜0.7本（サンプル制限で過小、実態はもう少し高い可能性）
- → 伸びてるアカ（reikan/fuchu）は**高頻度・定型量産**。これは自動生成基盤と相性が良い。

## 6. 3アカウント目への提言

1. **ニッチ**: 占い/スピ確定。`superuniverseoracle`の「宇宙/オラクル」路線はこの10アカに直接競合なし（差別化余地）。
2. **収益**: 物販アフィリ→**電話占い/占いアプリのASP案件**へ切替（fuchu型）。まず案件の有無・単価調査が必要。
3. **コンテンツ**: **「宇宙からのメッセージ＋絵文字置かせ」**でreikan型バズを狙う。12星座/今日の運勢を毎日定型量産（fuchu型）で自動生成に乗せる。
4. **人格**: 共感ストーリー型ペルソナ（実績数字＋身の上話）。
5. **投稿フロー**: メイン=占いコンテンツ→**リンクはリプライ**（既存の標準フローをそのまま流用）。
6. **頻度**: 1日4本以上の定型量産を前提に設計。
