# Issue #32 キューのGoogleスプレッドシート化

レビュー担当（Macユーザー）が**ブラウザでスプレッドシートに商品を追記**するだけで、
サーバーのバッチがそれを読んで記事を生成する仕組み。

## 方式：ウェブに公開（CSV）
Google APIのキーやサービスアカウントは**不要**。スプレッドシートをCSVとして公開し、
そのURLをバッチに読ませるだけ。中身は商品キーワードのみで機密ではない。

## セットアップ（管理者が一度だけ）

### 1. スプレッドシートを作成
新しいGoogleスプレッドシートを作り、1行目に**ヘッダー**を入れる（列名は固定）:

| brand | category | model_number | product_name | price | company_hint | url | affiliate_link_html |
|-------|----------|--------------|--------------|-------|--------------|-----|---------------------|

- 最低限 **brand と category**（または url）が入っていればOK。残りは空欄で可。
- 2行目以降にレビュー担当が商品を追記していく。

### 2. ウェブに公開（CSV）
1. メニュー **ファイル → 共有 → ウェブに公開**
2. 「リンク」タブ → 公開対象の**シート（タブ）**を選び、形式を **カンマ区切り形式（.csv）** に
3. **公開** → 表示されたURLをコピー（`.../pub?gid=0&single=true&output=csv` 形式）

### 3. サーバーに設定
`.env` に追記:
```
QUEUE_SHEET_CSV_URL=https://docs.google.com/spreadsheets/d/e/XXXX/pub?gid=0&single=true&output=csv
```
これでバッチ（`batch.py`）は自動的にこのシートを読む（未設定時は `data/queue.csv`）。

## 運用
- **レビュー担当**: シートに商品（ブランド・カテゴリ等）を追記するだけ
- **バッチ（毎朝cron）**: シートを読み、未生成の商品を生成→WP下書き
- **重複防止**: 既出商品（`articles_log.csv` のブランド+型番/カテゴリ）は自動スキップ。
  同じ商品を消さずに残しても二重生成されない。

## 動作確認
```bash
python batch.py --queue "https://docs.google.com/.../pub?...output=csv" --limit 1 --no-wp
```

## 制約・今後
- 公開CSVは**読み取り専用**。シートへの生成済みステータス書き戻しはしない
  （重複は articles_log.csv 側で防止）。書き戻しが必要なら将来サービスアカウント方式を検討（#4と統合）。
- 公開URLを知る第三者はシート内容（商品リスト）を読める。機密は入れないこと。
