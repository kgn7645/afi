# ロードマップ / WBS

アフィリエイト記事 自動化ツールの開発計画。進捗は GitHub の **Milestones（フェーズ）** と **Issues（作業項目）**、および **Projects（カンバン）** で管理する。

## 🎯 当面の目標
- **1日1時間程度の作業**で回る
- **月15万円程度**の収入
- **非属人化**（担当交代でも回る）
- 担当者は**スマホで記事チェックのみ**（確認→公開可否）
- **ほとんどの作業を自動化**

## ✅ 現状（完了済み = フェーズ0）
URL/手動入力 → 商品選定判定 → Gemini記事生成（「どこの国？」定型＋★信頼度評価）→ もしもリンク差し込み＋破損検証 → WordPress下書き投稿、までを単記事で自動化。簡易Web UI / CLI / スモークテスト付き。本番（ouchibase.com）で下書き投稿まで動作確認済み。

## 📊 ギャップ（目標との差分）
| 目標 | 現状 | 不足 |
|------|------|------|
| 1時間/日・チェックのみ | 1記事ずつ手動実行 | バッチ自動生成、スマホ承認運用 |
| ほぼ全自動 | 商品選定・リンク取得が手動 | 商品選定自動化、もしもリンク自動化 |
| 月15万円 | 投稿はできる | **集客（SEO/インデックス/高単価案件）** |
| 非属人化 | コード化済 | チェック標準化、監視、Runbook |

> ⚠️ **重要な前提**: 収益は「公開記事数」ではなく「検索流入」で決まる。記事生成の自動化は“量産”を可能にするが、**月15万円は SEO とインデックスの蓄積（数か月単位）で達成される**。M5（集客・収益最適化）が達成ドライバー。

## 🗺️ フェーズ（Milestones）と作業（Issues）

### M1. 量産パイプライン基盤
1日に複数記事を無人で下書きまで積む土台。
- #1 リンク再利用（ブランド/型番でリンク保存・自動差し込み）
- #8 もしもリンクHTMLのテンプレ自動生成（**全自動化の鍵**）
- #9 バッチ実行＆スケジューラ（1日N件 / cron）
- #10 重複商品/記事の検出・防止

### M2. 非属人チェック体制（スマホ承認）
担当者はスマホで確認→公開のみ。
- #11 WordPressモバイルアプリでのレビュー運用（**開発ゼロで実現可**）
- #13 記事チェック観点の標準チェックリスト
- #12 （任意）軽量スマホ承認UI

### M3. 商品選定の自動化
A作業を自動化し生成キューへ投入。
- #3 Amazon売れ筋からの商品候補 自動収集
- #14 候補→自動採否→生成キュー投入

### M4. 品質・正確性
チェック工数を下げ非属人化を支える。
- #15 企業情報のグラウンディング（「どこの国」誤生成対策）
- #16 自動QA（禁止表現/薬機法/事実整合/構成）

### M5. 集客・収益最適化（← 月15万円の達成ドライバー）
- #17 インデックス促進（Search Console / IndexNow / サイトマップ）
- #18 内部リンク自動化
- #19 高単価案件（成果報酬型）の自動差し込み
- #20 収益・流入モニタリング
- #4 生成実績のスプレッドシート書き戻し

### M6. 拡張機能
- #2 note 自動投稿 / #5 Canvaアイキャッチ / #7 ショート動画

### M7. 運用・非属人化
- #6 新環境セットアップ手順 / #21 常駐運用＋通知＋監視 / #22 運用Runbook

## 🔢 推奨実行順序
**M1 →（M2と並行）→ M4 → M3 → M5 →（随時）M6 / M7**

理由: まず「無人で下書きが積まれ（M1）、スマホで承認できる（M2）」状態を作ると、目標の運用形態が最短で立ち上がる。次に品質（M4）でチェックを軽くし、入口（M3）を自動化して完全無人化。収益の本丸（M5）は時間がかかるため早めに着手し並行で育てる。

## 📈 KPI（目安）
- 生成: 15記事/日（元シート目標）→ 下書き積み上げ
- チェック: 1記事あたり数分 × 担当のスマホ確認 = 合計1時間/日
- 収益: 流入と成果を M5 のモニタリングで追跡し、選定（M3）へフィードバック

## ⚙️ 進捗管理
- **Milestones** = フェーズM1〜M7
- **Issues** = 作業項目（WBSの葉）。`enhancement` / `documentation` でラベル分け
- **Projects** = カンバン（Todo / In Progress / Review / Done）で日次の進行を可視化
